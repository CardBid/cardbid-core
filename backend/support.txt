
Supportive notes

1. Doin anythingah port-related in configuraton docker+vpn will fail,
   turn off one of themah.

2. Enter cardbid_db PostgreSQL database inside docker container:
$  sudo docker exec -it cardbid_postgres psql -U admin -d cardbid_db
   use 'template1' instead of 'cardbid_db' to enter emergency database.

3. To set password of 'admin' postgres database user, use SQL on any database:
$  ALTER USER admin WITH PASSWORD 'adminpassword';

4. If 'cardbid_db' was dropped in means of debugging, create it in 'template1'
   database context and grant privileges afterwards:
$  GRANT ALL PRIVILEGES ON DATABASE cardbid_db TO admin;

5. To update djangah database with new 'models.py' stored inside 'x' directory:
$  python manage.py makemigrations x  # NOTE: x directory MUST exist and contain __init__.py
$  python manage.py migrate

6. In order to completely wipe out docker activity, and test it freshly do:
$  sudo docker compose down --rmi all -v
$  sudo docker compose up -d

7. Enter django shell:
$  docker compose exec backend python manage.py shell

8. Complete django JWT test
$  docker compose down -v
$  docker compose up --build -d
$  docker compose exec backend python manage.py makemigrations
$  docker compose exec backend python manage.py migrate
$  docker compose exec backend python manage.py create_test_users
   Now you would go to web browser and search:
   http://localhost:8000/api/token/
   there login with created user credentials, you should receive tokens,
   you can refresh token in
   http://localhost:8000/api/token/refresh/

9. Get user in django shell:
$  from django.contrib.auth import get_user_model
$  User = get_user_model()  # Should get CardbidUser
$  user = User.objects.get(email="dawtwilun@plm.com")


debug:
$ docker compose down && docker compose up -d
$ docker compose logs backend

musisz w tym katalogu (backend/core) zrobić sobie plik ".env" i w nim zdefiniować dwa klucze
publiczny i prywatny do stripe, oraz webhooka o tak:

STRIPE_SECRET_KEY=amogus
STRIPE_PUBLIC_KEY=sugoma
STRIPE_WEBHOOK_SECRET=patrzniżej

Wcześniej należy sobie odpalić webhooka stripe lokalnie (gdyż nie mamy jeszcze domeny
ani hostinga) poleceniem (które osobiście pobrałem aptem na linuxie):

$ stripe listen --forward-to localhost:8000/api/stripe-webhook/

Bo API endpoint do callbacków stripe jest "api/stripe-webhook/"
Po uruchomieniu polecenia on tam zwróci webhook secret i jego trzeba też do pliku ".env"
dodać. Po tym można dockera odpalić.

UWAGA!
Nie robić tak:

$ stripe trigger payment_intent.succeeded

Normalnie to wywali w logach serwera stripe błąd 500 (ustawiony ręcznie) bo prubuje sie
zakończyć transakcje która nie istnieje (w DB). Normalnie była by już jakaś zrobiona endpointem
API api/top-up i dopiero potem po akceptacji przez stripe api/stripe-webhook by otrzymał
dane. Sam działałby na stworzonym w tabeli Transaction rekordzie (przez top-up).
To i tak nic nie da bo ta komenda robi wogóle swój własny intent id!
